# SCENE/STITCH — RELEASE PLAYBOOK

## まず決めること

**SCENE/STITCH は、百科事典として公開しない。**

最初に出すべきものは「全部入りの歴史サイト」ではなく、
**読むだけで思想とレベル感が伝わる、少数精鋭の flagship archive** です。

世界トップレベルの質と、早めのリリースは両立できます。
ただし条件があります。

> **深さを全部のページに均等配分しないこと。**
> 代わりに、**少数の看板ページに極端に投資すること。**

## 品質の置き方

### world-class に見える条件

各 flagship 記事は、最低でも次を満たす。

1. **冒頭 3行で論点が立っている**
2. **音 / 服 / 都市 / 場所 / 影響** が一本につながっている
3. **ただの要約ではなく、視点がある**
4. **Key Tracks / Key Looks / Key Objects** がある
5. **参考資料が強い**
6. **記事単体でシェアしたくなる**

ここを満たせば、本数が少なくても十分に強いです。

### 逆に world-class に見えなくなる条件

- トップページだけ綺麗で中身が薄い
- 全カテゴリを用意したのに本文が浅い
- 記事ごとの型が揃っていない
- 参考資料が弱い
- 下書きレベルの記事を大量公開する

**重要:**
未完成のページが多いことより、
**完成度の低いページが公開されていること** のほうが痛いです。

---

## 公開の考え方

### 正しい出し方

**flagship first** で出す。

- 少数の看板記事を強くする
- その周辺は静かでよい
- Draft は見せてもいいが、目立たせすぎない
- 公開済みページの internal link を丁寧に張る

### 間違った出し方

- 10本揃うまで待つ
- Person / Brand / Graph / Search まで揃うまで待つ
- すべての記事を同じ密度で作ろうとする
- 「完成してから出す」という発想に入る

SCENE/STITCH は、完成品を一度だけ出すプロジェクトではなく、
**公開しながら強くしていく archive** として育てるのが正解です。

---

## 公開後の更新リズム

### 基本方針

**重い記事は少なく、軽い更新は多く。**

### おすすめの比率

- flagship scene essay : 1
- short note / mini essay : 3〜5

つまり、1本の大きい記事のあいだに、
短い考察や小さなアーカイブ記事を挟んでいく。

これなら質を落とさず、サイトが止まって見えない。

### short note の例

- なぜレコード店は文化の学校だったのか
- なぜ反ファッションは何度も流行するのか
- なぜ 1990s 東京は「編集」の都市だったのか
- ある一枚のジャケットは何を変えたのか
- ある一足のスニーカーはなぜ記号になったのか

## 成功の判断基準

最初の成功は PV の多さではない。

以下が起きれば勝ちです。

- 初見が 30 秒でテーマを理解できる
- 1本読んで「このサイトは視点が違う」と思われる
- 誰かが記事を保存したくなる
- 次の記事を待ちたくなる

SCENE/STITCH は、
**情報量の多さで勝つサイトではなく、文化の接続の鋭さで勝つサイト** です。

だから最初のリリースでは、
**本数より温度、網羅より編集、完成度より解像度** を優先してください。
